Técnicas e estratégias de testes

  • Teste é um conjunto de atividades que pode ser planejado antecipadamente e realizado sistematicamente.

  • É possível definir um “template” (esqueleto para teste).

  • Testes são um conjunto de passos ao qual é possível alocar técnicas de projeto e de estratégias específicas.

Objetivos do teste

  • O processo de teste, como qualquer outro deve ser revisto continuamente.

  • O objetivo é ampliar a atenção e possibilitar uma maior visibilidade e organização.

  • Resulta em uma maior agilidade e controle nos testes.

Fluxo de informações de testes

  • Depuração é a parte mais imprevisível do teste.

  • Um erro pode demorar um tempo indefinido para ser corrigido.

Teste de caixa preta

  • É um teste para verificar a saída dos dados usando entradas de vários tipos. (Inserir dados aleatórios).

  • Quanto mais testado, mais seguro é o teste.

  • São testes realizados nas entradas e saídas de dados.

  • Apenas a entrada correta resultará em um resultado correto.

Teste de caixa branca

  • Baseia-se nos testes de todos os caminhos lógicos e possíveis.

  • Apresentam um problema logístico com possibilidades infinitas. Mas é possível realizar um número limitado de testes.

  • Possui acesso ao código fonte, conhecendo a estrutura interna do produto.

Teste do caminho básico

  • Possibilita que o projetista defina uma média de complexidade de execução.

Complexidade Ciclomática

  • Valores que indicam a complexidade de um programa.

  • O valor computado define o número de caminhos do conjunto básico de um programa.

  • Oferece um limite máximo para o número de testes.

Para definir a complexidade ciclomática basta realizar a seguinte equação:

C = A - N + 2

C = complexidade.

A = Arestas (caminhos).

N = Nós (bolinhas).

No exemplo, seria: C = 11 - 9 + 2 = 4 caminhos
